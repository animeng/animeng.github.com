一、 提出问题：那些被浪费在“发布”上的时间

  2. 图片搬运：这是最头疼的部分。文章里引用的图片，需要手动从 Obsidian 附件目录复制到博客项目的 images 文件夹，还得小心翼翼地修改 Markdown 里的路径，一旦图片重名，更是灾难。
  3. 版本控制：最后还得手动打开终端，执行一连串的 git add, git commit, git push。

虽然我曾经尝试写过一些发布脚本，但脚本不仅难以维护，而且一旦环境发生变化（比如换了一台电脑），又得重新调试。发布一篇文章的心理负担，甚至盖过了写作本身的乐趣。
