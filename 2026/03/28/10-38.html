WebRTC 弱网优化面试速记卡

1. WebRTC 基本架构

1.1 核心流程

捕获 → 预处理 → 编码 → RTP打包 → SRTP加密 → 网络传输 → 接收解包 → 解码 → 播放

1.2 重要模块

  • Signaling：SDP Offer/Answer
  • ICE/STUN/TURN：连接建立与穿透
  • RTP/RTCP：音视频包传输及控制信息
  • Congestion Control：带宽估计与码率调整
  • Jitter Buffer：防抖动
  • FEC/NACK/RTX：丢包恢复
  • Opus/VP8/VP9/H.264/AV1：编码器

2. 弱网特征与影响

指标：

  • RTT（延时）
  • Packet Loss（丢包率）
  • Jitter（抖动）
  • Bandwidth Fluctuation（带宽波动）

表现：

  • 视频卡顿 / 花屏 / 模糊
  • 音频断续 / 延迟
  • A/V不同步
  • 通话中断

3. 弱网优化策略

3.1 网络连接优化

  • ICE 优先直连 + 快速切换
  • TURN relay 优化（QoS）
  • 保持 UDP NAT 映射（心跳）

3.2 链路质量探测

  • GCC：延迟趋势 + 丢包趋势
  • Transport-CC：精确包到达时间
  • REMB：接收端带宽估计

3.3 编码自适应

视频：

  • 动态码率（CBR / VBR）
  • 降分辨率：1080p→720p→480p
  • 降帧率：30fps→15fps
  • SVC 分层编码（Base layer on weak network）
  • 缩短关键帧间隔（Gop size）

音频：

  • Opus 自适应码率（6kbps–510kbps）
  • 内置 FEC
  • Narrowband→Wideband 切换
  • PLC（丢包隐藏）

3.4 丢包恢复

  • FEC：
    • FlexFEC 视频
    • Opus FEC 音频
  • NACK：重传丢失包
  • RTX：单独的重传流

3.5 播放端优化

  • JitterBuffer 动态调整
  • Audio/Video 同步调整
  • 延迟与卡顿的平衡

6. 高频面试题 & 思路

Q1: 弱网下 WebRTC 如何保证音视频质量？

A：

  1. 带宽探测（GCC / Transport-CC）
  2. 编码自适应（降码率 / 分辨率 / 帧率）
  3. 丢包恢复（FEC / NACK）
  4. JitterBuffer 调整
  5. 优化 ICE/TURN 链接

Q2: NACK 与 FEC 区别？

A：

  • NACK：丢包后重传，低延迟网络适用
  • FEC：预发冗余数据，高丢包+无法重传时适用
  • 可组合使用：音频Opus FEC + NACK

Q3: Opus 在弱网的优势？

A：

  • 广泛码率自适应能力
  • 内置 FEC 与 PLC
  • 窄带到全带宽无缝切换

Q4: WebRTC 带宽控制工作流程？

A：

  • 接收端估计带宽（REMB/Transport-CC）
  • RTCP 回传
  • 发送端调整编码参数

Q5: 如何调试弱网表现？

A：

  • getStats() 查看关键指标
  • webrtc-internals 曲线分析
  • netem 模拟丢包/延迟
  • 对比码率变化与分辨率调整
